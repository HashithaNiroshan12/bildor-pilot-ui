# 01 — Getting Started

> How to clone, install, and run the project for the first time.

---

## Q: What tools do I need installed on my machine before I start?

You need these installed globally:

| Tool | Minimum version | Check with |
|---|---|---|
| Node.js | 20.x or higher | `node --version` |
| npm | 10.x or higher | `npm --version` |
| Git | Any recent version | `git --version` |

If any are missing, install Node.js from [nodejs.org](https://nodejs.org) — it includes npm.

---

## Q: How do I set up the project for the first time?

```bash
# Step 1: Navigate into the project folder
cd "D:\Builder Pilot\bildor-pilot-ui"

# Step 2: Install all dependencies
npm install

# Step 3: Start Storybook (your visual development environment)
npm run storybook
```

Then open your browser at **http://localhost:6006**

---

## Q: What does `npm install` actually do?

It reads `package.json` and downloads all listed packages into the `node_modules/` folder.

You need to run it:
- The very first time you clone/open the project
- Any time someone adds a new dependency to `package.json`
- Any time `package-lock.json` changes

You do **not** need to run it every time you make a code change.

---

## Q: What is the difference between `npm install` and `npm ci`?

| `npm install` | `npm ci` |
|---|---|
| Use locally during development | Use in CI/CD pipelines (GitHub Actions) |
| Can update `package-lock.json` | Fails if lock file doesn't match |
| Slower but more flexible | Faster and more strict |

Always use `npm install` on your own machine. The GitHub Actions workflow uses `npm ci` automatically.

## Q: What are all the available commands?

| Command | What it does |
|---|---|
| `npm run storybook` | Start Storybook on localhost:6006 (your main dev tool) |
| `npm run build` | Build the library into `dist/` (JS + types) |
| `npm run typecheck` | Run TypeScript type checking without building |
| `npm run lint` | Check code style with ESLint |
| `npm run build-storybook` | Build a static Storybook for deployment |
| `npm run build:watch` | Auto-rebuild `dist/` on every save |

## Q: I ran `npm run storybook` but it says port 6006 is already in use. What do I do?

The `--ci` flag in the script suppresses the interactive prompt and Storybook will auto-pick the next available port (6007, 6008, etc.). Check the terminal output for the actual URL.

To always use a specific port, run directly:
```bash
npx storybook dev -p 6008 --ci
```

To find and stop what is using 6006 (on Windows PowerShell):
```powershell
netstat -ano | findstr :6006
# Note the PID from the last column, then:
taskkill /PID <pid-number> /F
```

## Q: What should I see when Storybook opens?

You should see a sidebar on the left with:
```
Components/
  Button
  Card
  Dropdown
  InputSwitch
```

Click any component to see it rendered. Each component has multiple **stories** (variants like "Primary", "Outlined", "Disabled").

The controls panel at the bottom lets you change props (like `label`, `disabled`, `severity`) live in the browser — no code changes needed.

---

## Q: I see a blank/white Storybook with no components listed. What's wrong?

Check these things in order:

1. **Stories not found** — Storybook looks for files matching `src/**/*.stories.tsx`. Make sure your story files follow that naming pattern.

2. **Import errors in preview.ts** — Open `.storybook/preview.ts`. If it imports a file that doesn't exist (like `primereact-overrides.css`), Storybook will fail silently. Check the file exists at `src/theme/primereact-overrides.css`.

3. **Check the browser console** — Press F12 → Console tab in the Storybook browser window for error messages.

## Q: What files should I NEVER manually edit?

| File/Folder | Why |
|---|---|
| `dist/` | Auto-generated by `npm run build`. Your changes will be overwritten. |
| `node_modules/` | Managed by npm. Never touch. |
| `package-lock.json` | Auto-managed by npm. Let npm update it. |
| `.storybook/vitest.setup.ts` | Auto-generated by Storybook. Leave it. |

---

## Q: What files will I edit most often as a developer on this project?

| File | When you edit it |
|---|---|
| `src/ui/SomeComponent.tsx` | Adding or changing a component's behaviour/styling |
| `src/ui/SomeComponent.stories.tsx` | Adding or updating visual test cases |
| `src/theme/tokens.css` | Changing brand colors, spacing, typography |
| `src/ui/index.ts` | Exporting a newly created component |
| `package.json` → `version` field | Before publishing a new release |
